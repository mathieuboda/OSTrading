# Plan: Architecture OpenClaw Trading System

## Contexte Projet
Le projet est en phase d'initialisation (Phase 1 non démarrée). SPEC FINALIZED, ROADMAP 4 phases. Architecture actuelle (`architecture.md`) trop superficielle (16 lignes). Aucun code écrit.

## Découverte Critique
**OpenClaw** (`openclaw/openclaw`, 250K+ stars GitHub) est un **assistant IA personnel multi-canal**, PAS un framework de trading dédié. L'article MindStudio décrit une architecture conceptuelle à construire **par-dessus** OpenClaw.

### Ce qu'OpenClaw fournit (infrastructure)
- Gateway multi-canal (Telegram, Slack, WhatsApp...)
- MCP (Model Context Protocol) pour lier le LLM aux APIs externes
- Cron jobs intégrés
- Système de skills modulaires (SOUL.md / SKILL.md)
- 13,700+ skills sur ClawHub marketplace

### Ce qu'on doit construire (logique trading)
- Agents Data / Strategy / Execution / Risk en Python
- Intégration Alpaca via MCP (`alpaca-mcp-server`)
- Indicateurs techniques (RSI, SMA)
- Backtest engine
- Logging structuré + healthchecks

## Références Techniques
- `ankit-aglawe/openclaw-lse-trading-agent` — architecture de référence: Python 3.12+, RSI/MACD/Bollinger, FinBERT sentiment, ATR stops, half-Kelly sizing, backtest, paper portfolio
- `jerrylearnscoding/claude-enterprise-openclaw-trading` — strategy trees, backtest engine, multi-agent orchestrator, circuit breakers
- Article MindStudio (11 avril 2026) — guide canonique de stratégie
- `alpaca-mcp-server` — bridge MCP vers Alpaca API

## Décisions Validées
- **Data Provider**: Alpaca Data API v2 (gratuit avec compte)
- **Stratégie initiale**: RSI Mean Reversion (RSI < 30 → Buy, RSI > 65 → Sell)
- **Notifications**: Telegram Bot API (déjà supporté par OpenClaw nativement)
- **Langage**: Python 3.12+ (via `uv` pour le packaging)
- **Courtier**: Alpaca Paper Trading (trade-only, pas de withdraw)

### Flux de Décision
```
[Cron trigger] → Data Agent (fetch prices via alpaca-py)
    → Strategy Agent (compute RSI, evaluate rules)
    → Risk Agent (check position limits, daily loss, drawdown)
    → [PASS] Execution Agent (place order via alpaca-mcp-server) → Alpaca Paper
    → [FAIL] Log rejection + notify Telegram
    → Log everything to JSON → Daily report
```

### Composants Clés
| Composant | Technologie | Rôle |
|-----------|------------|------|
| OpenClaw Gateway | Node.js/npm | Orchestration, cron, canaux, MCP |
| OpenClaw Skills | Python 3.12+ | Logique trading (indicateurs, stratégie, exécution, risque) |
| alpaca-mcp-server | MCP Server | Bridge standardisé LLM ↔ Alpaca API |
| alpaca-py | SDK Python | Accès direct aux données et ordres Alpaca |
| Telegram Bot | API native OpenClaw | Alertes temps réel |

  - JSON logs pour parsing

Notifications:
  - Telegram Bot API (intégration native OpenClaw)

  - python-dotenv (.env management)
